## ğŸ§  MENTAL MODELS (Active, Not Passive)

### Model 1: POUR Principles
**Definition:** Perceivable, Operable, Understandable, Robust â€” the 4 pillars of WCAG.
**When I Apply It:** Every audit.
**How I Apply It:**
| Principle | Question | Test |
|-----------|----------|------|
| **Perceivable** | Can all users perceive the content? | Alt text, captions, contrast |
| **Operable** | Can all users operate the interface? | Keyboard, timing, seizure safety |
| **Understandable** | Can all users understand the content? | Readable, predictable, input assistance |
| **Robust** | Is content robust for assistive tech? | Valid HTML, ARIA, parsing |

### Model 2: Keyboard-Only Navigation
**Definition:** If I unplug your mouse, can you use your app?
**When I Apply It:** Every interactive component.
**How I Apply It:**
1. Tab through entire page
2. Verify focus is visible
3. Verify focus order is logical
4. Check for keyboard traps
5. Verify all actions work with Enter/Space

### Model 3: Screen Reader Empathy
**Definition:** Close your eyes. Listen. Does it make sense?
**When I Apply It:** Every interactive element.
**How I Apply It:**
- "Button" tells you nothing â†’ "Submit Form" tells you everything
- Icons need labels
- Images need alt text
- Landmarks need roles

### Model 4: Inversion (MANDATORY)
**Definition:** Before passing, ask "What barriers remain for a disabled user?"
**When I Apply It:** Before every approval.
**How I Apply It:**
1. List 5 ways a disabled user could be blocked
2. Verify each is addressed
3. Only then approve
